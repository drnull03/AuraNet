Architecture diagram of AuraNet: a bootstrap job seeds a Kubernetes cluster. A centered control plane node (Cilium operator, SPIRE server, Hubble relay, and the AuraNet controller with its federated learning aggregator, zero trust controller, and autoheal components linked by NATS) sits above three worker nodes arranged side by side. Each worker node has Cilium agent, SPIRE agent, Envoy proxy, and an AuraNet agent running loader, runtime, and engine, where the engine contains the inference worker with three AI brains, the training worker, and the network worker. NATS buses fan out from the control plane to each worker's runtime and engine, and animated federated learning links fan out from the controller's FL aggregator to each worker's network worker. An eBPF enforcement layer sits at the edge of each worker node.
